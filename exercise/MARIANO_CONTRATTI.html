  1. Que es un sistema de control de versiones?

    Es un sistema que nos permite ir capturando nuestro trabajo en diferentes momentos. Podemos capturar o "tomar fotos" en varias instancias y obtener distintas versiones del mismo trabajo. No solo sirve para guardar nuestro proyecto individual, sino que también es una herramienta de trabajo colaborativo, ya que nos permite incorporar nuestro trabajo al trabajo de los demás colaboradores o viceversa. Una de sus virtudes es poder testear nuevas funcionalidades sin tener que hacerlo en el producto final utilizado por el cliente.

  2. Cual es la diferencia entre git y github?

    Git es el sistema de control de versiones propiamente dicho y GitHub es una plataforma donde podemos guardar todos nuestros repositorios, ver y colaborar con otros repositorios no propietarios, y utilizar las mismas funciones que podemos implementar en Git mediante la consola de comandos.
